# 省事部署：一步一步搞（Vercel 前端 + Render 后端）

不租服务器，用 **Vercel** 放前端、**Render** 放后端，全程在网页里点一点就能上线。

---

## 第一步：把代码放到 GitHub

1. 打开 [GitHub](https://github.com)，登录。
2. 点右上角 **+** → **New repository**。
3. 仓库名随便起，例如 `phone-competitor`，选 **Public**，**Create repository**。
4. 在本机打开 **phone-competitor** 所在文件夹（例如 `d:\competitor-monitor\phone-competitor`），在文件夹里打开 PowerShell 或 CMD，执行：

```bash
git init
git add .
git commit -m "init"
git branch -M main
git remote add origin https://github.com/你的用户名/phone-competitor.git
git push -u origin main
```

（把 `你的用户名/phone-competitor` 换成你刚建的仓库地址；若已存在 git，只执行 `git add .`、`git commit`、`git push` 即可。）

---

## 第二步：在 Render 部署后端

1. 打开 [https://render.com](https://render.com)，用 GitHub 登录。
2. 点 **Dashboard** → **New** → **Web Service**。
3. **Connect a repository**：选你刚推送的 `phone-competitor`（若没看到，点 **Configure account** 勾选该仓库权限）。
4. 配置：
   - **Name**：随便，如 `phone-api`。
   - **Region**：选离你近的（如 Singapore）。
   - **Root Directory**：若整个仓库就是 phone-competitor 里的内容，留空；若仓库是 competitor-monitor 且 phone-competitor 是子目录，填 `phone-competitor`。
   - **Runtime**：**Python 3**。
   - **Build Command**：  
     `pip install -r requirements.txt`
   - **Start Command**：  
     `uvicorn app.main_phone:app --host 0.0.0.0 --port $PORT`
5. 往下拉 **Environment Variables**，点 **Add environment variable**，先加两条：
   - **Key**：`CORS_ORIGINS`  
     **Value**：先填 `https://localhost:3000`（后面部署完前端再回来改成 Vercel 的域名）。
   - （可选）**Key**：`DEEPSEEK_API_KEY`，**Value**：你的 DeepSeek API Key（用竞品分析才需要）。
6. 点 **Create Web Service**，等几分钟构建和部署。
7. 部署成功后，顶部会有一个地址，例如：  
   `https://phone-api-xxxx.onrender.com`  
   **复制这个地址**，后面前端要用。

说明：免费版 Render 大约 15 分钟没人访问会休眠，下次打开会慢几秒；数据先用 SQLite，重启部署可能会清空，需要持久化看「第五步（可选）」。

---

## 第三步：在 Vercel 部署前端

1. 打开 [https://vercel.com](https://vercel.com)，用 GitHub 登录。
2. 点 **Add New** → **Project**，导入你同一个仓库 `phone-competitor`。
3. 配置：
   - **Root Directory**：若仓库根目录是 phone-competitor（里面有 `frontend` 文件夹），点 **Edit** 填 **frontend**；若仓库根是 competitor-monitor（里面有 phone-competitor），填 **phone-competitor/frontend**。保证构建时用的是有 package.json、next.config.js 的那一层。
   - **Framework Preset**：应自动识别为 Next.js。
   - **Environment Variables** 里点 **Add**：
     - **Name**：`NEXT_PUBLIC_API_URL`  
     - **Value**：填第二步里 Render 的地址，**不要带结尾斜杠**，例如：  
       `https://phone-api-xxxx.onrender.com`
4. 点 **Deploy**，等构建完成。
5. 部署成功后会有地址，例如：  
   `https://phone-competitor-xxx.vercel.app`  
   **复制这个地址**。

---

## 第四步：让前端能访问后端（改 CORS）

1. 回到 **Render** → 你的 Web Service → **Environment**。
2. 找到 **CORS_ORIGINS**，把值改成你的 **Vercel 地址**，例如：  
   `https://phone-competitor-xxx.vercel.app`  
   （若有多个域名用英文逗号分隔，不要空格。）
3. 点 **Save Changes**，Render 会自动重新部署（等一两分钟）。
4. 用浏览器打开你的 **Vercel 地址**，试一下：能打开页面、能进「产品数据库」、能点「下载产品模板」或「查询 GSMArena」，就说明前后端都通了。

若打开页面没问题，但点「下载产品模板」或列表一直转圈，多半是 CORS 没改对或没等 Render  redeploy 完成，检查 CORS_ORIGINS 是否和浏览器地址栏里的域名完全一致（含 `https://`）。

---

## 第五步（可选）：数据持久化 — 用 Render 的 PostgreSQL

默认用 SQLite，Render 每次重新部署可能清空数据。若希望数据一直保留，可以加一个免费 PostgreSQL 数据库：

1. 在 Render 的 **Dashboard** 点 **New** → **PostgreSQL**，建一个库（免费），记下 **Internal Database URL**（只在 Render 内部用，更稳）。
2. 在你的 **Web Service**（后端）→ **Environment** 里新增一条：
   - **Key**：`DATABASE_URL`  
   - **Value**：粘贴 Internal Database URL，形如：  
     `postgresql://user:pass@host/dbname`
3. **Save Changes**，等重新部署完成。
4. 之后产品、价格等数据会存在 PostgreSQL 里，不会因部署而丢失。

---

## 小结检查清单

- [ ] 代码已推到 GitHub。
- [ ] Render 上建了 Web Service，Build 用 `pip install -r requirements.txt`，Start 用 `uvicorn app.main_phone:app --host 0.0.0.0 --port $PORT`。
- [ ] Render 环境变量里设了 `CORS_ORIGINS` = 你的 Vercel 域名。
- [ ] Vercel 上导入了同一仓库，根目录指向 frontend（若需要），并设了 `NEXT_PUBLIC_API_URL` = Render 的地址。
- [ ] 浏览器打开 Vercel 链接能正常用；需要持久化时已加 PostgreSQL 的 `DATABASE_URL`。

按这个顺序做下来，就是「省事」的线上部署；以后改代码只要 `git push`，Render 和 Vercel 都会自动重新部署（若已开启自动部署）。
