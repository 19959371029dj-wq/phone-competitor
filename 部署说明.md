# 手机竞品分析工具 - 线上部署说明

本说明适用于在一台 **Linux 服务器（VPS）** 上部署前后端，通过 Nginx 对外提供访问。也简要说明「仅部署后端 + 前端用 Vercel」的方式。

---

## 一、服务器要求

- 系统：Ubuntu 22.04 / Debian 11+ 等（x64）
- 内存：建议 ≥ 1GB
- 已安装：Python 3.10+、Node.js 18+、Nginx

---

## 二、部署步骤概览

1. 把项目上传到服务器（如 `/opt/phone-competitor`）
2. 配置后端环境变量并常驻运行（systemd）
3. 构建前端并配置 `NEXT_PUBLIC_API_URL` 指向后端
4. 用 Nginx 做反向代理，并可选配置 HTTPS

---

## 三、后端部署

### 3.1 目录与依赖

```bash
cd /opt/phone-competitor
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 3.2 环境变量

在项目根目录创建 `.env`（或由 systemd 注入）：

```bash
# 数据库（默认 SQLite，数据在 ./data/ 下）
DATABASE_URL=sqlite:///./data/phone_competitor.db

# 线上前端地址，用于 CORS（多个用逗号分隔）
CORS_ORIGINS=https://your-domain.com,https://www.your-domain.com

# 若使用 DeepSeek 竞品分析，填写 API Key
DEEPSEEK_API_KEY=sk-xxx

# 可选：GSMArena 请求间隔（秒），避免 429
GSMARENA_REQUEST_DELAY_SECONDS=25
```

说明：`CORS_ORIGINS` 必须包含你实际访问前端的域名（含协议），否则浏览器会报跨域错误。

### 3.3 用 systemd 常驻运行

创建 `/etc/systemd/system/phone-competitor.service`：

```ini
[Unit]
Description=手机竞品分析 API
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/phone-competitor
EnvironmentFile=/opt/phone-competitor/.env
ExecStart=/opt/phone-competitor/venv/bin/uvicorn app.main_phone:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

然后：

```bash
sudo systemctl daemon-reload
sudo systemctl enable phone-competitor
sudo systemctl start phone-competitor
sudo systemctl status phone-competitor
```

后端只监听 `127.0.0.1:8000`，由 Nginx 反向代理到外网。

## 四、前端部署（同机）

### 4.1 构建

```bash
cd /opt/phone-competitor/frontend

# 指定线上 API 地址（替换为你的后端域名或同一域名下的 /api 代理）
export NEXT_PUBLIC_API_URL=https://api.your-domain.com
# 若前后端同域且用 Nginx 把 /api 代理到 8000，可设为空或当前站点的 /api，例如：
# export NEXT_PUBLIC_API_URL=

npm ci
npm run build
```

若使用「同域 + Nginx 代理 /api」见下节，则 `NEXT_PUBLIC_API_URL` 可设为 `https://your-domain.com`（与前端同域），由 Nginx 把 `/api` 转到后端。

### 4.2 运行（生产）

```bash
npm run start
```

或也用 systemd 管理（端口 3000，仅本机）：

```ini
# /etc/systemd/system/phone-competitor-frontend.service
[Unit]
Description=手机竞品分析前端
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/phone-competitor/frontend
Environment=NODE_ENV=production
Environment=PORT=3000
ExecStart=/usr/bin/npm run start
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

## 五、Nginx 反向代理（前后端同机示例）

假设域名：`https://your-domain.com`，前端 3000，后端 8000。

```nginx
server {
    listen 80;
    server_name your-domain.com;
    # 若已配置 HTTPS，可在此做 80 -> 301 跳转

    # 前端（Next.js）
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 后端 API（与前端同域时可这样写，前端 NEXT_PUBLIC_API_URL 填 https://your-domain.com）
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        client_max_body_size 20M;
    }

    # 若 API 文档也要对外
    location /docs {
        proxy_pass http://127.0.0.1:8000/docs;
        proxy_set_header Host $host;
    }
}
```

若后端用独立子域（如 `https://api.your-domain.com`），则：

- 前端构建时：`NEXT_PUBLIC_API_URL=https://api.your-domain.com`
- 后端 `.env`：`CORS_ORIGINS=https://your-domain.com`
- Nginx 为 `api.your-domain.com` 单独写一个 `server`，`proxy_pass` 到 `http://127.0.0.1:8000`

## 六、HTTPS（推荐）

使用 Let’s Encrypt：

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

按提示选择为上述 `server_name` 申请证书，certbot 会自动改 Nginx 配置并续期。

## 七、前后端分离部署（前端 Vercel、后端 VPS）

- **后端**：按「三」在 VPS 上部署，并保证有公网域名（如 `https://api.your-domain.com`），Nginx 只代理该域名到 `127.0.0.1:8000`。
- **前端**：在 Vercel 导入仓库，构建目录选 `frontend`，在 Vercel 环境变量里设置：
  - `NEXT_PUBLIC_API_URL=https://api.your-domain.com`
- **后端 .env**：`CORS_ORIGINS=https://你的Vercel域名.vercel.app,https://your-domain.com`（把实际访问前端的域名都写上）。

这样浏览器访问 Vercel 上的前端，请求会发到你的 VPS 后端。

---

## 八、常见问题

| 问题 | 处理 |
|------|------|
| 前端报跨域错误 | 检查后端 `CORS_ORIGINS` 是否包含前端访问的完整域名（含 `https://`） |
| 接口 404 | 确认 Nginx `location /api/` 的 `proxy_pass` 写对，且后端在 8000 端口正常 |
| 上传/下载失败 | Nginx 增加 `client_max_body_size 20M;` |
| 数据库丢失 | SQLite 库在 `./data/phone_competitor.db`，备份该目录；换机时一起拷贝 |

## 九、简要检查清单

- [ ] 服务器已安装 Python 3.10+、Node 18+、Nginx
- [ ] 后端 `.env` 中 `CORS_ORIGINS` 为实际前端域名
- [ ] 前端构建时设置了 `NEXT_PUBLIC_API_URL`
- [ ] systemd 已 enable/start 后端（及可选的前端）
- [ ] Nginx 已配置并 `nginx -t` 通过，且已重载
- [ ] 若需 HTTPS，已用 certbot 配置并测试访问

按上述步骤即可完成线上部署。若你使用 Docker 或其它平台，可在此基础上再包一层容器或平台配置。
